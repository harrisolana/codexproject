# 个人交易提醒系统开发需求规格书

## 1. 项目定位

本项目是一个个人交易决策辅助网站。系统只做数据监控、条件判断、决策进度展示和提醒，不接入真实交易，不自动下单。

核心目标：

- 监控多个交易对的实时行情。
- 支持多数据源扩展。
- 支持自定义交易条件。
- 支持多个条件组合成一个交易决策。
- 当决策条件满足时提醒用户手动判断是否下单。
- 用户可以点开决策，查看每个条件的当前值、目标值、状态和达成进度。

## 2. 第一阶段边界

第一阶段只做提醒系统骨架，不做真实交易。

包含：

- 行情看板。
- 决策列表。
- 决策详情。
- 条件引擎。
- 页面提醒。
- 提醒历史。
- 本地配置存储。

不包含：

- 自动下单。
- 交易所 API Key 管理。
- 用户登录。
- 云端同步。
- 真实 X 舆论数据接入。
- 真实链上数据接入。
- 回测系统。

## 3. 核心架构

```text
数据源层 Data Sources
  ↓
标准化数据层 Normalized Context
  ↓
指标与信号层 Indicators & Signals
  ↓
条件引擎 Condition Engine
  ↓
决策引擎 Decision Engine
  ↓
提醒与复盘 Alerts & Journal
```

设计原则：

- 数据源可替换。
- 条件类型可扩展。
- 决策触发过程可解释。
- 提醒只辅助判断，不代替用户下单。
- 后续接入 X 舆论和链上数据时不推翻现有结构。

## 7. 条件引擎模块

条件是系统中的最小判断单元。每个条件独立计算，并返回可解释结果。

条件状态：

- `met`：已满足。
- `near`：接近满足。
- `unmet`：未满足。
- `insufficient_data`：数据不足。
- `source_error`：数据源异常。
- `disabled`：已停用。

条件返回结构：

```js
{
  id: "price-below-045",
  type: "price_below",
  status: "met",
  currentValue: 0.448,
  targetValue: 0.45,
  progress: 1,
  distanceText: "已低于目标价 0.44%",
  message: "价格条件已满足",
  updatedAt: 1710000000000
}
```

第一版条件类型：

- 价格大于某值。
- 价格小于某值。
- 区间涨幅大于某值。
- 区间跌幅大于某值。
- 成交量大于过去均量的 N 倍。
- 价格上穿 MA。
- 价格下穿 MA。
- RSI 小于某值。
- RSI 大于某值。

后续条件类型：

- X 舆论热度超过阈值。
- X 情绪评分转正。
- 某关键词提及量增加。
- 链上大额转入。
- 链上大额转出。
- 指定地址发生买入。
- 交易所净流出增加。
- Funding Rate 低于阈值。
- Open Interest 增加。

## 8. 决策引擎模块

一个决策由多个条件组成。系统根据触发规则判断是否提醒用户。

决策结构：

```js
{
  id: "wld-dip-buy",
  name: "WLD 低吸提醒",
  symbol: "WLDUSDT",
  enabled: true,
  triggerMode: "all",
  cooldownMinutes: 30,
  conditions: [
    {
      id: "price-below-045",
      type: "price_below",
      source: "binance",
      timeframe: "1m",
      required: true,
      params: { value: 0.45 }
    },
    {
      id: "rsi-below-35",
      type: "rsi_below",
      source: "binance",
      timeframe: "15m",
      required: true,
      params: { period: 14, value: 35 }
    }
  ]
}
```

第一版触发模式：

- 全部条件满足。
- 必须条件全部满足，且可选条件满足 N 个。

后续触发模式：

- 任意 N/M 个条件满足。
- 条件分组。
- 权重评分。
- 多时间周期确认。
- 冷却时间内不重复提醒。

## 9. 决策进度模块

决策卡片展示：

- 决策名称。
- 交易对。
- 触发进度，例如 `3/5`。
- 当前状态。
- 最近更新时间。
- 是否启用。
- 是否处于冷却时间。

决策详情展示：

- 每个条件名称。
- 条件来源。
- 当前值。
- 目标值。
- 达成进度。
- 满足状态。
- 数据更新时间。
- 条件解释文本。

示例：

```text
WLD 低吸提醒
进度：3/5
状态：接近触发

价格 <= 0.45
当前 0.452，还差 0.44%

RSI < 35
当前 38.2，还差 3.2

成交量 >= 1.8x
当前 1.6x，接近满足
```

## 15. 第一阶段 MVP 验收标准

第一阶段完成后应满足：

- 用户可以打开网站看到行情看板。
- 用户可以看到至少 2 个示例决策。
- 每个决策能显示条件满足进度。
- 点开决策能看到每个条件的当前值、目标值和解释。
- 条件满足时能触发页面提醒。
- 提醒能进入历史记录。
- 页面在桌面端和移动端布局正常。
- 代码结构预留数据源、条件、决策、提醒模块。

## 16. 风险与注意事项

数据风险：

- 第三方行情接口可能限流。
- 数据源可能延迟。
- 不同交易所价格可能不同。
- X 舆论和链上数据需要处理噪声。

交易风险：

- 系统只做提醒，不代表确定性信号。
- 条件满足不等于一定应该下单。
- 用户需要保留人工判断。
- 需要记录复盘，避免过度依赖提示。

技术风险：

- 纯前端直接请求 API 可能遇到网络限制。
- 多数据源会带来时间对齐问题。
- 链上数据可能需要索引服务或第三方 API。
- 情绪数据需要防止被单一热点误导。

## 17. 下一步开发建议

下一步建议基于现有行情看板实现第一阶段 MVP：

1. 增加决策列表区域。
2. 抽象条件引擎。
3. 增加 2-3 个内置示例决策。
4. 增加决策详情抽屉。
5. 显示每个条件的达成进度。
6. 条件满足时触发页面提醒。
7. 将提醒写入本地历史记录。

先把骨架做稳，再接入 X 舆论、链上数据和更多外部数据源。
